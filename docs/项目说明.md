# 学科助教系统 — 项目说明文档

## 1. 项目概述

### 1.1 项目背景
本项目响应"挑战杯"竞赛"面向一流学科建设的学科垂类大模型与创新应用开发"赛道要求，
聚焦**助教（教师端）**场景，基于大语言模型（LLM）和检索增强生成（RAG）技术，
构建智能教学辅助系统，覆盖备课、批改、学情分析、知识库管理等教学核心环节。

### 1.2 项目定位
- **目标用户**：高校教师
- **核心价值**：以 AI 赋能教学全流程，实现减负增效
- **技术路线**：LLM + RAG + 向量知识库
- **差异化优势**：融合学科知识库的 RAG 增强生成，保证内容专业可信

## 2. 技术方案

### 2.1 整体架构

```
用户界面 (React + Ant Design)
        ↕ HTTP RESTful API
业务服务层 (FastAPI)
        ↕
核心引擎层
  ├── RAG 引擎 (检索增强生成)
  ├── LLM 接口 (OpenAI 兼容)
  └── 向量知识库 (ChromaDB)
```

### 2.2 RAG 实现方案
```
用户查询
    ↓
语义检索知识库 (ChromaDB + BGE Embedding)
    ↓
获取 Top-K 相关文档片段
    ↓
构建增强提示词 (原问题 + 参考文档 + 角色设定)
    ↓
调用 LLM 生成回答
    ↓
返回结构化结果
```

### 2.3 关键技术组件

| 组件 | 选型 | 说明 |
|------|------|------|
| Web 框架 | FastAPI | 高性能异步框架，自动生成 OpenAPI 文档 |
| 向量数据库 | ChromaDB | 开源轻量，支持持久化，余弦相似度检索 |
| 嵌入模型 | BGE-small-zh-v1.5 | 中文优化，512维，兼顾效果与性能 |
| LLM 接口 | OpenAI 兼容 | 支持讯飞星火、DeepSeek、OpenAI 等 |
| 文档处理 | pypdf + 滑动窗口分块 | PDF 解析，500字块+50字重叠 |
| 前端框架 | React 18 + TypeScript | 类型安全，组件化开发 |
| UI 组件 | Ant Design 5 | 企业级 UI，支持中文本地化 |
| 数据可视化 | 内嵌组件 | 雷达图、趋势图、进度展示 |

## 3. 功能模块详述

### 3.1 智能备课

**输入**：课程名称 + 章节 + 教材内容（可选） + 课时数 + 要求

**处理流程**：
1. 从学科知识库检索相关教材内容（RAG）
2. 调用 LLM 生成结构化教案
3. 解析 JSON 并构建完整教案对象

**输出**：
- 三维教学目标（知识/能力/素质）
- 教学方法与资源列表
- 分课时教学流程（BOPPPS 模式）
- 每个活动的时长、类型、内容
- 教师活动与学生活动描述
- 课后作业与思考题

**降级方案**：LLM 调用失败时返回基础模板教案

### 3.2 作业批改与辅导

**智能批改流程**：
1. 接收题目、学生答案、参考答案
2. 构建专业批改提示词（角色设定 + 批改标准）
3. LLM 评分并生成详细反馈
4. 解析结构化结果

**批改输出**：
- 得分与得分率
- 综合评语
- 优点列表
- 不足与错误列表
- 改进建议
- 涉及知识点
- 逐句/逐步详细分析

**练习题生成流程**：
1. 接收课程/知识点/难度参数
2. LLM 生成多种题型
3. 附加答案与详细解析

### 3.3 学情精准洞察

**个体分析流程**：
1. 导入学生成绩记录（可 CSV 粘贴）
2. LLM 分析各知识点掌握度
3. 识别薄弱与优势知识点
4. 生成趋势分析与预警
5. 输出个性化学习建议

**分析维度**：
- 综合评分
- 知识点掌握度雷达
- 薄弱/优势识别
- 成绩变化趋势
- 学习预警（持续下降/严重薄弱）
- 个性化学习建议
- 是否需要重点关注

**班级分析**：
- 汇总统计（平均分/及格率/优秀率）
- 分数段分布
- 共性薄弱环节排名
- 重点关注名单

### 3.4 学科知识库

**功能**：
- 教材/讲义 PDF 导入
- 自动文本分块（500字块+50字重叠）
- 向量化存储（ChromaDB）
- 语义相似度检索
- 多课程独立管理

**技术要点**：
- 使用滑动窗口分块保证上下文连贯
- 存储元数据（来源、章节、索引位置）
- 余弦距离计算语义相似度
- 支持按课程过滤检索范围

## 5. 部署指南

### 5.1 开发环境

```bash
# 后端
cd backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
# 配置 .env 文件
uvicorn app.main:app --reload --port 8000

# 前端
cd frontend
npm install
npm run dev
```

### 5.2 生产环境
```bash
# 后端
uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4

# 前端构建
cd frontend
npm run build
# 产出在 dist/ 目录，部署至 Nginx/CDN
```

### 5.3 环境变量
```env
LLM_API_KEY=sk-xxx
LLM_BASE_URL=https://api.openai.com/v1
LLM_MODEL_NAME=gpt-4o-mini
EMBEDDING_MODEL_NAME=BAAI/bge-small-zh-v1.5
VECTOR_DB_PATH=./knowledge_base/vector_store
HOST=0.0.0.0
PORT=8000
```
